## 计算机硬件底层：从「电」到「能算题的电脑」（从 0 到 1）

计算机**底层只有一件事**：

**用电的「通 / 断」，表示数字「1/0」，再用一堆开关搭出能计算、能记忆、能控制的机器。**

### 1）最底层物理：为什么只能是「二进制」？

现实世界里，电子设备**最稳定、最简单**的状态只有两个：

- 有电 / 高电压 → 表示 **1**
- 没电 / 低电压 → 表示 **0**

这就是**二进制**的来源：

不是数学家随便选的，是**硬件物理限制逼出来的**。

所有计算、文字、图片、视频，底层全是一串 1/0。

### 2）硬件最小单元：晶体管 = 一个可控开关

现代计算机的**最小零件**是**晶体管**（Transistor）。

你可以把它理解成：

**用电压控制的电子开关**。

- 给控制端加电 → 开关闭合 → 通电 = 1
- 不给电 → 开关断开 → 断电 = 0

现在一颗 CPU 里有**上百亿个**这种小开关。

### 3）基础积木：逻辑门（用晶体管搭出来）

把几个晶体管连在一起，就能做出**逻辑门**，这是计算机能 “思考” 的起点：

- **非门（NOT）**：输入 1→输出 0，输入 0→输出 1
- **与门（AND）**：两个都为 1 才输出 1
- **或门（OR）**：只要一个为 1 就输出 1

所有**加减乘除、判断、跳转**，底层全是逻辑门在算 1 和 0。

### 4）能算数了：加法器 → ALU（算术逻辑单元）

把逻辑门串起来，就能做出**加法器**。

- 加法有了 → 减法（加负数）
- 减法有了 → 乘法（多次加）
- 乘法有了 → 除法（多次减）

能做算术 + 逻辑判断（>、<、==）的模块，叫 **ALU**，是 **CPU 的计算心脏**。

### 5）指挥中心：控制单元 CU + 时钟

光会算不行，还要知道**先算什么、数据从哪来、结果放哪去**。

- **控制单元（CU）**：翻译指令，指挥整个硬件干活

- 时钟（Clock）：给全机统一 “节拍”

  3GHz = 每秒钟跳 30 亿次，每一跳完成一步操作

ALU + CU + 寄存器 = **CPU 核心**。

### 6）记忆模块：从快到慢的三层存储

计算机需要 “记住数据和指令”，硬件按**速度 / 成本**分成三层：

1. **寄存器**（在 CPU 里）：最快，容量极小
2. **内存（RAM）**：断电丢失，速度中等，临时存程序
3. **硬盘 / SSD**：断电不丢，速度慢，永久存文件

底层原理：

- 内存：用电容充放电存 1/0
- SSD：用电子俘获存 1/0（浮栅晶体管）

### 7）连接所有零件：总线（Bus）= 硬件高速公路

CPU、内存、显卡、硬盘不能各玩各的，靠**总线**连起来：

- **数据总线**：传 1/0
- **地址总线**：指定 “我要读写哪个位置”
- **控制总线**：发命令（读 / 写 / 中断）

### 8）人机交互：输入 & 输出

- 输入：键盘 / 鼠标 → 转成电信号 → 总线 → CPU
- 输出：CPU → 总线 → 显卡 / 屏幕 → 转成光信号给你看

### 9）总结硬件底层

**硬件底层**：用电的通断表示 1/0，用晶体管做开关，用逻辑门做运算，用总线连所有部件。

**CPU 本质**：就是一个高速、自动的二进制运算指挥中心

**计算机 = 百亿个电子开关，按固定节拍，对二进制做逻辑运算，并用总线和存储协作的机器。**



## 电脑的画面是如何呈现的？

**现在的图片、视频、游戏画面，在电脑眼里全是什么样的二进制。**

你只要记住一句话：

**人眼看到的是「画面」，电脑眼里只有「一堆 0 和 1 组成的数字表」**

### 1）先搞懂：一张「图片」怎么变成二进制？

#### ① 画面 = 无数个小方块 → 像素

任何屏幕图像，本质都是**密密麻麻的小光点**，叫 **像素（Pixel）**。

- 1920×1080 = 横向 1920 个、纵向 1080 个像素
- 一张图 = 约 207 万个小光点

电脑不认识 “图”，只认识：

**这一行第几个光点、亮什么颜色**。

#### ② 每个颜色 = 一组二进制数字

电脑用 **RGB** 描述颜色：

- R：红
- G：绿
- B：蓝

每个颜色用 **8 位二进制** 表示（0~255）：

- 8 位二进制 = 00000000 ~ 11111111
- 对应十进制：0 ~ 255

#### 举几个立刻能懂的例子

- 纯黑：

  R=0，G=0，B=0

  二进制：

  `00000000 00000000 00000000`

  

- 纯白：

  R=255，G=255，B=255

  二进制：

  `11111111 11111111 11111111`

  

- 纯红：

  R=255，G=0，B=0

  二进制：

  `11111111 00000000 00000000`

  

**一个像素 = 24 位二进制**

**一张图片 = 几百万个像素的二进制串**



### 2）屏幕怎么把「二进制」变回「画面」？

1. **显卡显存**

   里面存着一整块数据：

   `第1行第1个像素: 11111111 00000000 00000000`

   `第1行第2个像素: ...`

   全是 0 和 1。

   

2. **GPU（显卡核心）**

   它不干别的，就**高速算每个像素应该是什么 0/1**。

   

3. **显示器**

   收到二进制信号 → 控制每个小点

   - 1 = 亮 / 数值大 = 更亮
   - 0 = 不亮 / 数值小 = 更暗

   

你看到的画面，就是**显示器在高速朗读一份巨大的二进制表格**。



### 3）游戏画面 = 「实时算出来的二进制动画」

游戏**不是存好的视频**，是 CPU + GPU **每秒重新算几十次全新的二进制画面**。

游戏里的一切，全是二进制

#### ① 3D 模型 = 二进制坐标

人物、房子、山，在电脑里都是：

- 一堆顶点坐标（x,y,z）

  比如：x=120, y=35, z=80

  二进制：

  ```
  1111000 100011 1010000
  ```

- 顶点之间怎么连，也是二进制记录

#### ② 皮肤 / 贴图 = 上面说的「图片二进制」

把贴图的 0/1 贴到模型表面。

#### ③ 你的操作 = 二进制指令

- 你按 W → 键盘传给电脑 `00000001`
- CPU 算：角色往前移动多少 → 新坐标二进制
- 把新坐标发给 GPU

#### ④ GPU 做最累的活：3D → 2D 屏幕像素

GPU 干的事：

1. 把 3D 模型二进制坐标
2. 加上光影二进制计算
3. 转成**屏幕上每个像素的 RGB 二进制**
4. 塞进显存
5. 显示器立刻显示

#### ⑤ 动起来 = 每秒刷 60/144 次二进制

- 60 帧 = 每秒生成 **60 张全新的二进制画面**

- 144 帧 = 144 次

  你看到的 “流畅”，就是**电脑疯狂刷新 0 和 1 表**。

### 4）连声音也是二进制

游戏里的枪声、脚步声：

- 声音波形 → 按时间采样 → 变成二进制数字
- 声卡把二进制 → 转成电流 → 喇叭震动发声

### 5）终极极简总结

1. **图像 = 像素的 RGB 二进制清单**
2. **屏幕 = 照着二进制清单发光的板子**
3. **游戏 = CPU+GPU 每秒算几十万次新的二进制像素表，刷到屏幕上**
4. **底层全一样**：晶体管处理 0/1 → 总线传 0/1 → 显存存 0/1 → 显示器转成光
